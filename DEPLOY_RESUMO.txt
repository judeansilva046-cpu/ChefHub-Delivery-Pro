1️⃣  Criar projeto
    → https://supabase.com
    → New Project
    → Name: ChefHub Delivery Pro
    → Region: Brazil (São Paulo) 🇧🇷
    → Plano: Free (suficiente)
    ⏳ Aguardar criação (~2 min)

2️⃣  Executar SQL
    → Supabase Studio → SQL Editor
    → New Query
    → Colar: supabase/migrations/001_init_schema.sql
    → Run
    ✅ Esperar 12 tabelas serem criadas

3️⃣  Copiar credenciais
    → Settings → API

    COPIAR ESTES 2 VALORES (IMPORTANTES!):

    NEXT_PUBLIC_SUPABASE_URL = https://xxx.supabase.co
    NEXT_PUBLIC_SUPABASE_ANON_KEY = eyJhbGc...

    → Continue with GitHub
    → Autorizar

    → Procurar: ChefHub Delivery Pro
    → Import

3️⃣  Adicionar variáveis (OBRIGATÓRIO!)

    NEXT_PUBLIC_SUPABASE_ANON_KEY = [colar valor do Supabase]

4️⃣  Deploy!
    → Clicar no botão grande "Deploy"
    → Aguardar 3-5 minutos

    🔄 Vercel vai:
       ✅ Clonar repo
       ✅ Build
       ✅ Deploy

    ✅ Quando aparecer "Congratulations!"
       Seu app está ONLINE! 🎉

❌ Build failed no Vercel?
   → Verificar logs no Vercel Dashboard
   → Testar localmente: npm run build
   → Corrigir erro e fazer git push

❌ Supabase connection error?
   → Verificar NEXT_PUBLIC_SUPABASE_URL
   → Verificar NEXT_PUBLIC_SUPABASE_ANON_KEY
   → Vercel → Settings → Environment Variables
   → Redeploy

❌ Página em branco?
   → F12 → Console → verificar erros
   → Checar Network requests
   → Verificar Supabase logs

❌ Muito lento?
   → Vercel Analytics → verificar latência
   → Supabase → verificar queries
   → Adicionar índices nas tabelas
